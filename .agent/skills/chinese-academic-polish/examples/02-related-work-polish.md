# 示例 2：相关工作段落润色

## 场景

用户要求润色 `latex/sections/zh/02_related.tex` 中"新兴采集技术与最接近先例"小节的工业先例段落。

## 输入（原文）

```latex
与本文最接近的工业实践包括：（1）Envoy 代理的 AccessLog 过滤器支持按路由或状态码配置日志输出策略，但仅作用于代理自身的 access log，不驱动下游微服务的应用日志采集；（2）Kong API 网关的 File-log/TCP-log 插件可按路由启用日志记录，但属于静态配置，不具备基于流量统计的动态 URL 模式生成能力；（3）OpenResty 生态的 \texttt{lua-resty-logger-socket} 等模块实现了 Lua 层的动态日志控制，但动态策略局限于网关节点自身，未形成跨节点的应用日志定向采集闭环。
```

## 问题诊断

1. **超长段落**：三个工业先例挤在一段里，整段 200+ 字无换行。
2. **并列编号不透气**：用分号连接的编号列表让人喘不过气。
3. **尾巴太长**：每个先例的"但……"从句超过 40 字。

## 输出（润色后）

```latex
与本文最接近的工业实践有三类。其一，Envoy 代理的 AccessLog 过滤器支持按路由或状态码配置日志输出策略，但仅作用于代理自身的 access log，无法驱动下游微服务的应用日志采集。其二，Kong API 网关的 File-log/TCP-log 插件可按路由启用日志记录，但属于静态配置，不具备基于流量统计动态生成 URL 模式的能力。其三，OpenResty 生态的 \texttt{lua-resty-logger-socket} 等模块实现了 Lua 层的动态日志控制，但策略局限于网关节点自身，未形成跨节点的采集闭环。
```

## 改动说明

| 改动 | 手法 |
|------|------|
| "包括：（1）……；（2）……；（3）……" → "有三类。其一，……。其二，……。其三，……" | 模式 E：分段叙述 |
| 每个先例用句号结尾，独立成句 | 模式 A：拆长句 |
| "不驱动" → "无法驱动"，更口语 | 模式 B：去模板词 |
| "未形成跨节点的应用日志定向采集闭环" → "未形成跨节点的采集闭环" | 去冗余修饰 |
