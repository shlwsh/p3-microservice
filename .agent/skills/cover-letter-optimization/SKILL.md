# Goal

撰写或优化一封**规范、专业、精炼**的中文学术投稿信，确保控制在一页 A4 纸内，结构完整、语言自然、数据准确。

## Instructions

### 第一步：收集论文信息

在开始撰写之前，从用户提供的论文内容或 LaTeX 源码中提取以下信息：

- **论文标题**
- **作者名单与通讯作者**（含单位和邮箱）
- **目标期刊名称**
- **核心研究问题**（领域痛点是什么）
- **主要创新贡献**（3–5 项，每项需包含技术手段和与已有工作的区别）
- **关键实验结果**（量化数据，如降幅百分比、性能指标）
- **可复现性说明**（实验脚本、数据是否公开）

如果用户未提供完整信息，从论文摘要（`00_abstract.tex`）、引言（`01_intro.tex`）和实验（`06_experiments.tex`）中自动提取。

### 第二步：撰写投稿信

#### 结构（五段式，行内标签）

每段用 `\textbf{标签}\quad` 开头，**不使用** `\subsection*`，节省纵向空间。

| 段落 | 字数 | 核心内容 |
|------|------|----------|
| **研究背景** | 80–120 字 | 领域痛点 → 传统方法不足 → 本研究切入点 |
| **主要贡献** | 200–300 字 | 编号分列，每项独立成段 |
| **实验验证** | 80–120 字 | 实验设置 + 量化结果 + 可复现性 |
| **期刊契合** | 40–60 字 | 属于哪个方向 + 与期刊定位的关系 |
| **声明** | 40–60 字 | 原创、未发表、未一稿多投、作者同意、感谢 |

#### 贡献分列写法（三段式）

每项贡献遵循：**做了什么 → 怎么做的 → 和已有方案的区别**。

```latex
（N）\textbf{贡献名称}。一句话说做了什么。
接着一到两句说具体技术手段。
最后一句点出与已有方案的区别。
```

### 第三步：篇幅控制（一页 A4）

LaTeX 排版参数：

```latex
\documentclass[11pt,a4paper]{ctexart}
\geometry{left=2.5cm,right=2.5cm,top=2.2cm,bottom=2.2cm}
\setlength{\parskip}{0.3em}
\setlength{\parindent}{2em}
```

控制手段（按优先级排列）：
1. 字号 11pt，不用 12pt。
2. 贡献使用行内编号（1）（2）…，不用 `enumerate`。
3. 签名区合并为两行 `tabular`。
4. 如果仍超一页：优先压缩"研究背景"和"期刊契合"，贡献和实验数据不删。

### 第四步：中文润色

#### 禁用表达替换表

| 禁用 | 替换 |
|------|------|
| 综上所述 | 从上述分析可以看出 / 直接省略 |
| 此外 | 同时 / 另外 / 直接省略 |
| 值得注意的是 | 删除，直接陈述 |
| 形成本质区别 | 有本质不同 |
| 制约了…效率 | 拖慢了…效率 |
| 造成…显著压力 | 让…都吃紧 |
| 实现…的自适应 | …跟着实际流量走 |

#### 短句规则

- 每句不超过 30 个汉字（不含引用标记和公式）。
- 长句按"主语—谓语—宾语"拆成 2–3 个短句。
- 列举超过 3 项用分号隔开。

#### 动词选择

- 用"把""让""跟着"等日常动词替代"实现""进行""开展"等官腔动词。
- 保留"提出""设计""构建""验证"等学术核心动词。
- 数据表述保持精确（`降幅 98.4%`），不说"大幅降低"。

### 第五步：编译与版本化输出

1. 编译投稿信 LaTeX：

```bash
cd submission && xelatex cover_letter.tex && xelatex cover_letter.tex
```

2. 确认 PDF 为 1 页。

3. 按版本化命名输出到 `docs/`：

```bash
MAX_V=$(ls -1 docs/ | grep -oP '^v\K[0-9]+(?=-)' | sort -n | tail -1)
VERSION=${MAX_V:-1}
TS=$(date +%Y%m%d-%H%M%S)
cp submission/cover_letter.pdf "docs/v${VERSION}-投稿信-${TS}.pdf"
```

4. 清理旧的无版本号文件。

5. 同步更新 `submission.tar.gz`。

### 第六步：自检清单

编译前逐项确认：

- [ ] 全文控制在一页 A4 纸内？
- [ ] 五个段落齐全（背景、贡献、验证、契合、声明）？
- [ ] 贡献分列为独立段落，每项有加粗标题？
- [ ] 关键实验数据（降幅、CPU、节点数、重复次数）完整？
- [ ] 消除了"综上所述""此外""值得注意的是"等模板词？
- [ ] 每句话不超过 30 个汉字？
- [ ] 通讯作者、单位、邮箱、日期正确？
- [ ] 输出文件名带版本号和时间戳？

## Constraints

- 投稿信**必须一页 A4 纸**，超出则按优先级压缩。
- 所有实验数据必须与论文正文一致，不得编造或模糊化。
- 不使用"首次""创新性地""开创性"等自我评价词汇，除非有充分客观依据。
- 文件名遵循 `doc-naming-convention` 技能的规范。
- 本技能仅适用于中文学术投稿信；英文 Cover Letter 需另行处理。
