# 示例 1：创建新的设计报告文档

## 场景说明
用户希望在 `docs-zh` 目录下输出一个关于系统前端的架构设计方案。

## 输入请求
"请在 docs-zh 下生成一个前端架构设计方案报告。"

## 执行过程
1. 检测到输出路径为 `docs-zh`，且输出类型为“架构设计方案”（属于方案/设计报告），触发 `doc-naming-convention` 技能。
2. 提取当前日期，假设为 `20260611`。
3. 提取初始版本号 `v1.0`。
4. 提取文档短描述 `前端架构设计方案`。
5. 自动组装输出文件名为：`docs-zh/v1.0_20260611_前端架构设计方案.md`。

## 预期输出（文档内容头部）

```markdown
# 前端架构设计方案

**版本：** v1.0
**日期：** 2026年6月11日

## 1. 概述
本文档描述了系统前端部分的架构方案...
```
