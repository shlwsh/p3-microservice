# 示例 3：参数不完整时的交互

## 场景描述

用户要求创建新技能，但未提供必要的参数信息。

## 输入

```text
用户: 创建一个新技能。
```

## 执行过程

1. **检测参数缺失**：`skill-name`、`description`、`goal` 均未提供
2. **向用户询问必填信息**

## 预期输出

```text
需要以下信息来创建技能：

1. **技能名称**（skill-name）：小写字母 + 中划线格式，如 `my-skill`
2. **技能描述**（description）：该技能的适用场景和触发条件
3. **技能目标**（goal）：一句话说明要实现的最终目标

可选配置：
- 是否需要 scripts/ 目录？（默认：否）
- 是否需要 resources/ 目录？（默认：否）
- 存放级别：项目级（默认）还是全局级？
```

## 用户补充后

```text
用户: 名称是 log-analyzer，用于分析服务端日志文件并生成报告，目标是快速定位错误和异常。
```

**继续执行正常创建流程。**
