# paper1-multi-agent-review — 可移植技能包

多角色模拟同行评审（预检 → 盲审 → 交互质询 → PI 综合 → 落盘报告）。**本目录自包含全部文档**；当前默认配置为 **p3-microservice** 分布式日志采集论文。

## 目录文件

| 文件 | 用途 |
|------|------|
| **SKILL.md** | Cursor 入口（Agent 必读） |
| **软件学报投稿要点.md** | **JOS 官网投稿指南 + 敬告作者（2026）** |
| **顶级期刊论文规范指南.md** | **评审要求、文档结构、行文、图表公式、参考文献五维规范** |
| **审核细则.md** | **各角色 Phase 1–2 勾选清单（效率核心）** |
| **config.md** | p3 路径、作者、L0 位置 |
| **config.example.md** | 移植到新项目时的配置模板 |
| **reference.md** | 输出清单、模板、议题簇 |
| **设计方案.md** | Phase 0–5 完整流程与 DoD |
| **角色定义手册.md** | 七角色职责、语气、高效路径 |
| **参考文献归档细则.md** | **C6 正文引用文献核实与 papers/ 下载** |
| **已知问题清单.md** | 预检 C1–C6 + **J1–J7（JOS）** + **T1–T5（顶刊）** |
| **论文领域要点.md** | SQ、基线、代码锚点 |
| **改稿衔接.md** | 审核后改稿阶段概要 |

## 安装到其他项目

```bash
mkdir -p .cursor/skills
cp -r /path/to/paper1-multi-agent-review .cursor/skills/
```

1. 编辑 `config.md`、`已知问题清单.md`、`论文领域要点.md`、`审核细则.md`（按新项目定制）。  
2. 在 Cursor 中说：**「执行论文多角色审核」** 或 @ `paper1-multi-agent-review`。

## 触发语

- 执行 p3 论文多角色审核 / **软件学报投稿前审核**
- 模拟审稿人审一遍论文  
- 全面评审建议 / 论文审核  
- 按顶级期刊规范检查论文结构、行文、图表公式与参考文献

## 产出位置

```
reviews/{YYYYMMDD-HHmmss}/
├── 00-上下文清单.md … 05-全面评审报告.md
├── 行动清单.md
└── 执行摘要.md
```

## 会话沉淀要点（为何需要审核细则）

| 漏检类型 | 强化位置 |
|----------|----------|
| 表1 σ ±1000 | 审核细则 §0、§4；C3a |
| 首期 vs 三期表述 | 审核细则 §5；C3c |
| 2s vs 400ms 公平性 | 审核细则 §2、§7 CL-FAIR |
| 仿真消融 vs 实测 | 审核细则 §2 M5、§3 |
| 基金/作者占位 | 审核细则 §1 C5、§5 E1/E6 |
| **JOS 仅引外文** | 软件学报投稿要点 §3.2；J3 **P0** |
| **JOS 创新对比不足** | J2；DOMAIN-J* |
| **JOS 定稿/在线投稿** | 审核细则 §10；05 附录 |
| **顶刊 RQ/贡献链路缺失** | 顶级期刊论文规范指南；T1、CL-TOP-RQ |
| **图表公式著录不规范** | T4/T5；CL-TOP-FMT |
