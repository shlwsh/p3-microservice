# 参考文献归档细则

> **规则**：论文正文 `\cite{}` 的**每一条**文献必须可核实、有效，并归档至 [`data/papers/`](../data/papers/)（仓库根 [`papers/`](../papers/) 为符号链接）。  
> **负责角色**：Phase 1 📚 预检 + Phase 2 📐 伦理（可审计性）。

---

## 1. 通过标准

| 层级 | 要求 |
|------|------|
| **核实** | 每条 cited 条目 DOI 在 CrossRef/CHNDOI 可解析，或 URL 可访问（HTTP 200） |
| **归档** | `archive_status` 为 `ok`（PDF）或 `snapshot`（官网 HTML / CrossRef JSON / 书籍目录页） |
| **禁止** | bib 中 cited 键缺失；**无效 DOI**（如 doi.org 404）；无归档文件 |
| **清单** | `data/papers/cited_papers_manifest.json` 与 cited 数量一致且全部通过 |

---

## 2. 自动化命令（审核必跑）

```bash
# 核实 + 下载（约 1–2 分钟，需网络）
python3 scripts/verify_cited_papers.py --download

# 仅核实 DOI/URL（不下载）
python3 scripts/verify_cited_papers.py
```

**退出码 0** = 可投稿/可合并；**非 0** = 存在无法核实或未归档条目。

---

## 3. 归档类型说明

| archive_status | 含义 | 典型路径 |
|----------------|------|----------|
| `ok` | 开放 PDF | `data/papers/2024_Tingting_wang2024rca.pdf` |
| `snapshot` | 官网 HTML（中文期刊/技术文档） | `data/papers/web_snapshots/limingshu2019logmgmt.html` |
| `snapshot` | CrossRef JSON（闭源期刊 DOI 已验证） | `data/papers/web_snapshots/soldani2024logs_crossref.json` |
| `snapshot` | 书籍/Open Library 页 | `data/papers/web_snapshots/turnbull2014monitoring.html` |

**说明**：闭源期刊（Springer/ACM/Wiley）不要求必须取得 PDF，但**必须**有 CrossRef/CHNDOI 核实 + 元数据快照。

---

## 4. 预检检查项 C6 / L1

| ID | 严重性 | 检查 | 验收 |
|----|--------|------|------|
| C6a | **P0** | cited 键均在 `references.bib` | 脚本无「bib 中缺失」 |
| C6b | **P0** | 全部 DOI/URL 有效 | manifest 无 `doi_valid:false` |
| C6c | **P0** | 全部已归档 | manifest `paywall` 计数 = 0 |
| C6d | P1 | 新增 `\cite` 后重跑脚本 | `cited_papers_manifest.json` 时间戳更新 |

---

## 5. 新增文献工作流（作者）

1. 在 `latex/references.bib` 添加条目（**优先真实 DOI**；中文期刊用 CNKI/CHNDOI 可查 DOI）
2. 正文 `\cite{key}` 后运行：`python3 scripts/verify_cited_papers.py --download`
3. 若失败：修正 DOI/URL，或更换为可核实文献；**勿保留无效 DOI**
4. 提交时包含 `data/papers/` 下新增 PDF/快照（体积过大时仅提交 manifest + 快照，PDF 可 LFS）

## 7. 审核产出

```text
cited: 18 | ok(PDF): 4 | snapshot: 14 | paywall: 0 | 脚本: exit 0
```
