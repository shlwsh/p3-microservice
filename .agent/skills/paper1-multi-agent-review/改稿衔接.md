# 审核完成后的改稿衔接

> **所属技能包**: `paper1-multi-agent-review`  
> **前置**: 已完成 Phase 0→5，`reviews/{run_id}/` 下 12 文件齐全  
> **本文档目的**: 定义从评审报告到论文修改的结构化桥接流程

---

## 1. 改稿启动流程

用户触发改稿时（如「按行动清单改稿」），执行 Agent 按以下步骤启动：

```text
Step 1  读取最近一次 reviews/{run_id}/行动清单.md
Step 2  读取 04-PI综合裁决.md 确认 PI 决定和优先级
Step 3  若用户给出编辑/基准论文意见，先读取 基准论文对照与改稿闭环.md
Step 4  在 docs/ 生成 YYYYMMDD-HHMMSS-重大修改改进方案.md
Step 5  按 P0→P1→P2 顺序逐项处理
Step 6  每完成一个 AP，更新行动清单状态 [ ] → [x]
Step 7  编译 PDF 并另存 docs/v{N}-论文稿件-jos-YYYYMMDD-HHMMSS.pdf
Step 8  生成 docs/YYYYMMDD-HHMMSS-二次评审报告.md
Step 9  全部完成后生成 Point-by-Point Response Letter
```

## 2. 修改阶段定义

| 阶段 | 目标 | 涉及 AP | 建议扮演 | 验收方式 |
|------|------|---------|----------|----------|
| **A. 结构重组** | 合并实验子节、重写 Discussion、移除 FAQ、移附录 | P0 结构类 | 📚 博士生 + ✍️ 编辑 | 页数 ≤ 目标期刊限制 |
| **B. 数据一致性** | 统一 Abstract/CoverLetter/Table 数值 | P0 数据类 (C3) | 📚 + 📊 统计 | grep 验证关键数字一致 |
| **C. 内容深化** | 精炼贡献、补 SOTA 对比、深化 Limitation | P1 类 | 📚 博士生 | 审稿人必答问题全覆盖 |
| **D. 语言润色** | 英文表达、术语一致、引用格式 | P2 类 | ✍️ 编辑 | 无系统性语言问题 |
| **E. 合规补全** | AI 声明、CRediT、Cover Letter | P2 伦理类 | ✍️ + 📐 伦理 | 检查表全勾 |
| **F. 版本化与二评** | 时间戳方案、编译版本、二次评审 | 改稿闭环类 | 🎯 + ✍️ + 📊 | `docs/` 产物齐全 |

## 3. 逐条修改协议

对行动清单中的每个 AP，执行 Agent 须：

```markdown
### AP-{ID}: {描述}

**原始问题**: {来自评审报告的问题描述}
**关联角色**: {来源角色标记}
**涉及文件**:
- 主修改: `sections/{file}.tex` L{start}-L{end}
- 需同步: `00_abstract.tex`、`table_ii.tex` 等

**修改内容**:
```diff
- 原始文本...
+ 修改后文本...
```

**验证**:
- [ ] 编译通过 (`latexmk -pdf main.tex`)
- [ ] 交叉引用完整 (`grep '\\ref{' | 检查 label 存在`)
- [ ] 数值与 JSON/CSV 一致
```

## 4. 多文件同步要求

修改任何一处涉及数值的内容时，必须同步检查并更新以下位置：

| 数值类型 | 需同步的文件 |
|----------|-------------|
| 实验指标 (M1/M2/τ) | `00_abstract.tex`、`05_2_main_results.tex`、`table_ii.tex`、`zh/` 对应文件、Cover Letter |
| 基线名称/描述 | `02_related.tex`、`05_1_setup.tex`、`table_*.tex` |
| 贡献点表述 | `01_intro.tex`、`00_abstract.tex`、`06_conclusion.tex` |

## 5. Point-by-Point Response Letter 模板

改稿完成后，生成以下格式的回复信：

```markdown
# Point-by-Point Response to Reviewers

> Paper: {论文标题}
> Revision based on: reviews/{run_id}/

---

## Reviewer A (Methodology)

### Comment A-M1: {问题标题}
**Original comment**: {原始评审意见}

**Response**: {回复，含修改说明}

**Changes made**:
- `sections/xx.tex` L{nn}: {具体修改描述}
- Highlighted in \textcolor{blue}{blue} in the revised manuscript

---
(对每条 Major/Minor 重复)
```

## 6. 改稿后自检清单

完成所有修改后，在提交前自检：

| # | 检查项 | 验证方式 |
|---|--------|----------|
| 1 | 行动清单中 P0 全部 [x] | 检查 `行动清单.md` |
| 2 | 编译无 error | `latexmk -pdf main.tex` |
| 3 | 无 Overfull hbox 超过 10pt | 检查 `.log` 文件 |
| 4 | Abstract 数值与 Table II 一致 | grep 对比 |
| 5 | 中英文版本数值一致 | 对比 `table_ii.tex` vs `zh/table_ii.tex` |
| 6 | 所有 `\ref{}` 有对应 `\label{}` | grep 交叉检查 |
| 7 | Cover Letter 已更新 | 检查 submission/ |
| 8 | Response Letter 已生成 | 检查 reviews/{run_id}/ |
| 9 | 改进方案带时间戳 | `ls docs/*重大修改改进方案.md` |
| 10 | 编译稿件带版本号 | `ls docs/v*-论文稿件-jos-*.pdf` |
| 11 | 二次评审报告带时间戳 | `ls docs/*二次评审报告.md` |

## 7. 改稿后回归审核

改稿完成后建议：**新建 `run_id` 再跑一轮本技能**（Phase 0–5），重点验证：

1. 上一轮 P0 是否全部关闭
2. 修改是否引入新问题
3. 生成差异对比报告（见 [回归验证.md](回归验证.md)）

若用户要求保留历史稿，禁止删除旧 `docs/v4-论文稿件*.pdf` 或旧版本时间戳 PDF；新编译结果可以覆盖标准路径 `docs/v4-论文稿件-jos.pdf`，但必须同时另存一个带版本号与时间戳的副本。
