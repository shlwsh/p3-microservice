## 基线定义（对比公平性）

| 模式 | 行为 | 配置锚点 | 审核要点 |
|------|------|----------|----------|
| **定向采集** | 按 Redis 关注清单过滤后上报 | `rule_matcher.go`、`collector.go` | 日志源 **2s** 间隔 |
| **全量采集** | 全级别日志上报 Loki | 同架构，关闭/绕过清单 | 日志源 **400ms** 间隔 |
| **公平性** | 同 Docker Compose、同 8 服务 | `docker-compose*.yml` | **必须披露** 发射频率差异 |

审稿人常问：98.4% 中有多少来自「少发日志」vs「清单过滤」？→ 讨论段须边界化表述。

## 三次转换（E2E 叙事）

```
网关日志 → Center 解析/聚合 → Redis 关注清单
         → Agent 拉清单 + 本地匹配 → 定向上报 → Loki
```

**代码锚点**：

| 转换 | 路径 |
|------|------|
| 1 | `center/pkg/parser/`、`center/pkg/strategy/triple_transform.go` |
| 2 | Redis 清单 schema / Center 写入 |
| 3 | `agent/pkg/matcher/rule_matcher.go`、`agent/pkg/collector/collector.go` |

方法论角色须能指出：论文算法描述 ↔ 上述文件一致。
