# 《软件学报》投稿要点（2026 有效）

> **来源**：官网投稿指南与「敬告作者」（https://www.jos.org.cn/ → 作者投稿 / 下载区）。  
> **技能用途**：多角色审核时，以本文为 **JOS 初审/退稿高发区** 的权威对照；与 [审核细则.md](审核细则.md) 配合使用。  
> **本仓库稿源**：`latex/main-jos.tex`（rjthesis）；模板见 `docs/latex-models/software-journal/`。

---

## 0. 与审核技能的映射

| 官网要求类别 | 主责角色 | 预检/ Issue ID |
|--------------|----------|----------------|
| 原创、无一稿多投 | 📐 伦理 | J1 / ETHICS-J* |
| 创新性 + 同行对比（含国内） | 🔬 领域 | J2 / DOMAIN-J* |
| 文献调研充分、中文文献 | 🔬 领域 | J3 / C4-JOS |
| 方法有证明或充分实验 | 🔍 方法论 | J4 / METHOD-J* |
| 非产品报告/低技术含量 | 🔬 + ✍️ | J5 |
| 格式、模板、图表、摘要 | ✍️ 编辑 | J6 / EDIT-J* |
| 投稿流程、声明、定稿 | 📐 + 作者 | J7（审核外清单） |

**触发 JOS 专审**：用户指定「投软件学报」或 `TARGET_JOURNAL=JOS` 时，Phase 1 追加 **J1–J7**；Phase 2 各角色读本文对应节。

## 1. 基本投稿要求（必须严格遵守）

### 1.1 原创性与独家性

- 必须是**未在任何正式出版物发表**的原创论文。
- **不允许一稿多投**（含会议论文集有正式书号）。
- **不支持翻译稿**。

**审核动作（📐）**：稿内勿暗示「已发表/已投他刊」；结论与摘要勿写「本文首次在 XX 会议报告」除非作者确认未正式出版。

### 1.2 学术道德

- 严禁抄袭、剽窃、侵权；违规将通知单位且不再接受第一作者投稿。

**审核动作（📐）**：引用与 `references.bib` 一致；图表若为项目自有需可溯源；AI 声明完整（见 H1）。

### 1.3 投稿方式与定稿

- **仅官网在线投稿**：https://www.jos.org.cn/ →「作者投稿」。
- 不支持邮件、打印稿；上传前查病毒；**清稿定稿**（投稿后系统**不允许更新**）。
- 格式：**优先 Word**（官网模板）；接受 LaTeX（上传 PS），**录用后需转 Word**。
- **初审通过后**：全体作者签字《投稿声明》（官网下载区）。
- **审稿周期**：一般 6 个月内；超 6 个月（经确认）可自行处理。

**审核动作（✍️ + 作者）**：

| 项 | 投稿前 | 审核是否覆盖 |
|----|--------|--------------|
| 清稿定稿 | 一次上传即终稿 | ✍️ 报 EDIT-J-FINAL「定稿前须过 P0」 |
| Word 模板 | 官网下载 | 本仓库 LaTeX 对齐 rjthesis；录用后转 Word 为作者事项 |
| 投稿声明 | 初审后 | 记入 `05` 附录「投稿外 checklist」 |

### 1.4 其他

- 联系方式准确；可选「前沿观点」「Fast Track」等栏目（与本稿关系：一般投**研究论文**）。

## 2. 写作与内容核心要求

| 要求 | 审核要点 |
|------|----------|
| 结构完整、论点明确、论据可靠、论证合理、层次分明、可读性强 | ✍️ IMRaD；🔍 论据↔实验 |
| 结合国家需求、国际前沿、**深入系统**研究 | 🔬 引言动机；勿写成浅层应用 |
| **中英文摘要、关键词**完整（中文稿亦要英文摘要） | ✍️ `main-jos.tex` 中英摘要齐全 |
| 参考文献**查新、查全、明确标引** | 🔬 C4-JOS |
| **必须包括并比较国内同类工作** | 🔬 **J3 硬性**；仅外文易被退 |

## 3. 官网「敬告作者」— 初审/退稿高发区

### 3.1 创新性不足（**最常见退稿**）

| 表现 | 审核检查 |
|------|----------|
| 未与国内外同类工作**详细、充分比较** | 🔬 §2 相关工作是否有**对比表/段落**；非仅罗列 |
| 尤其缺少**国内文献**对比 | 🔬 中文文献 ≥ 建议 **30%** 或 ≥ **8 篇**（工程稿） |
| 像「工作报告」「读书笔记」，缺原创贡献 | 🔬 贡献(1)(2)(3) 是否可验证、是否超越工程记录 |

**Issue 模板**：`DOMAIN-J1: 创新点未与国内定向日志/可观测性工作充分对比`

### 3.2 文献调研不充分

| 表现 | 审核检查 |
|------|----------|
| 未介绍领域近年国内外发展现状 | 🔬 引言+相关工作是否含 2020–2026 脉络 |
| 仅引国外、忽略国内同行 | J3：`references.bib` 中文条目占比 |

**建议国内方向**（p3 稿）：微服务日志、云原生监控、AIOps、日志采样、国产栈可观测性、《软件学报》《计算机学报》相关系统论文。

### 3.3 技术/方法缺乏说服力

| 表现 | 审核检查 |
|------|----------|
| 新算法/系统无数学证明或**充分实验对比** | 🔍 算法形式化 + 表1/图4；禁「明显优越」无数据 |
| 大型系统仅设想、无实现或验证 | 🔍 phase3 实测 + 代码路径 |
| 成熟技术堆叠、无明显创新 | 🔬 三次转换/动态清单是否成体系贡献 |

### 3.4 内容不适合本刊

| 表现 | 审核检查 |
|------|----------|
| 高级科普、产品介绍、广告、纯观点 | ✍️ grep「本产品/领先/唯一」等营销语 |
| 软件应用到其他领域但**软件技术含量低** | 🔬 核心是否为采集**方法与系统** |
| 他刊退稿未本质修改重投 | 📐 作者确认（审核不臆测） |

### 3.5 格式与规范

| 表现 | 审核检查 |
|------|----------|
| 未用官方模板 | ✍️ `main-jos.tex` + `docs/latex-models/software-journal/` |
| 图表不清晰、无图序图题图注 | ✍️ 每个 figure/table 有 `\caption` + 正文引用 |
| 参考文献标引不规范、不完整 | ✍️ C4 + 国标/学报格式 |
| 文字不精炼、语法排版错误 | ✍️ 摘要字数、长句 |

### 3.6 其他

- 一稿多投或已正式发表 → J1  
- 投稿后想改 → 须先终止审理再重投（作者流程，记入投稿外 checklist）

## 4. 投稿前实用检查清单（可写入 `05` 附录）

### 4.1 内容与学术（审核 Phase 1–2 覆盖）

- [ ] **创新点比较段**写得足够长、有表格或分项对比（国内外 + 国内 ≥ 阈值）
- [ ] 参考文献含**足够中文文献**，正文有**明确对比论述**（非仅列在 bib）
- [ ] 中英文摘要、关键词完整且与正文一致
- [ ] 实验/实现支撑主要结论（非仅架构图）
- [ ] 公平性、局限、规模边界已写清（防「工作报告」印象）

### 4.2 格式与模板（✍️ 主查）

- [ ] 使用官网或本仓库 **rjthesis** 对齐模板编译 `main-jos.tex`
- [ ] 图表清晰、有图序图题图注、正文逐一引用
- [ ] 参考文献格式统一、近 5 年文献占合理比例
- [ ] 无基金占位、AI 声明、作者信息准确

### 4.3 投稿流程（作者执行，审核仅提醒）

- [ ] 全体作者确认：**无一稿多投、无学术不端**
- [ ] 官网注册、在线上传**清稿**（LaTeX → PS/PDF 按系统要求）
- [ ] 备好《投稿声明》待初审通过后签字
- [ ] 阅读官网《稿件处理流程》PDF

### 4.4 自动化辅助命令

```bash
# JOS 主稿存在
test -f latex/main-jos.tex && echo OK main-jos

# 中英文摘要（rjthesis 常见关键字）
grep -n "abstract\|摘要\|keywords\|关键词" latex/main-jos.tex | head -20

# 中文文献粗查（期刊名/语言字段/中文标题字符）
python3 - <<'PY'
import re, pathlib
bib = pathlib.Path("latex/references.bib").read_text(encoding="utf-8", errors="ignore")
entries = re.split(r"@\w+\{", bib)[1:]
zh_keys = []
for e in entries:
    key = e.split(",", 1)[0].strip()
    block = e.lower()
    if re.search(r"[\u4e00-\u9fff]", e):
        zh_keys.append(key)
    elif any(x in block for x in ("软件学报", "计算机学报", "中文", "journal of software", "journal of computer")):
        zh_keys.append(key)
print("entries:", len(entries), "zh-ish:", len(zh_keys))
print("sample:", zh_keys[:10])
PY

# 营销/产品腔
grep -Ein "本产品|国内领先|国际领先|唯一|填补空白" latex/sections/zh/ docs/v4-论文稿件.md || true

# 图表 caption
grep -c '\\caption' latex/sections/zh/*.tex
grep -c '\\begin{figure}' latex/sections/zh/*.tex
```

## 5. 各角色 JOS 增量必答（摘要）

| 角色 | JOS 增量 3 问 |
|------|----------------|
| 🔬 | ① 国内同类工作是否引且比？② 创新是否超越「工程记录」？③ 近 5 年脉络是否完整？ |
| 🔍 | ① 主张是否有实验/形式化支撑？② 系统是否已实现并验证？③ 对比是否公平披露？ |
| ✍️ | ① 是否 rjthesis/官网体例？② 中英摘要关键词？③ 图表题注与引用？ |
| 📐 | ① 无一稿多投表述风险？② AI/原创声明？③ 可复现材料是否可公开？ |
| 📊 | ① 实验是否「充分」支撑 JOS 口径（非仅 1 组表）？② n=3 是否说明探索性？ |
| 🎓 PI | 若 J2/J3 不通过 → 倾向 **Major Revision 或拒稿建议**，即使数值无误 |
