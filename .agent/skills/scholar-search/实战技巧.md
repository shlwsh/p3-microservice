## 1. 双流水线（必记）

| 流水线 | 工具 | 对象 | 产出 |
|--------|------|------|------|
| **A. 调研扩展** | `scholar_search.py` + `download_papers.py` | 主题相关、待引入文献 | `data/scholar/*.json`、`papers/*.pdf` |

## 2. 搜索技巧（p3 / 微服务日志方向）

### 2.1 英文：OpenAlex + 相关性关键词

宽泛查询易返回「高引但不相关」论文。推荐：

```bash
python .agent/skills/scholar-search/scripts/scholar_search.py \
  --query "microservice observability log collection sampling" \
  --backend openalex --year-from 2020 --num 20 \
  --relevance-keywords "microservice" "log" "Loki" "Promtail" "tracing" "AIOps" "gateway" "sidecar" \
  --arxiv-only \
  --output data/scholar/results_obs.json --bibtex data/scholar/refs_obs.bib
```

**子主题分拆搜索**（合并去重后质量更高）：

| 子主题 | 查询示例 | 关键词 |
|--------|----------|--------|
| 根因/诊断 | `root cause analysis microservice logs` | RCA, cascade, multimodal |
| 可观测性综述 | `microservice observability survey industrial` | tracing, metrics, log |
| 云原生 AIOps | `AIOps cloud platform log anomaly` | AIOps, IT operations |
| Sidecar/网格 | `service mesh sidecar overhead` | mesh, sidecar, latency |

### 2.2 中文：《软件学报》/《计算机学报》/ CRAD

OpenAlex 对中文期刊覆盖弱，**中文文献须人工从官网核对 DOI**：

| 期刊 | DOI 前缀 | 核实入口 |
|------|----------|----------|
| 《软件学报》 | `10.13328/j.cnki.jos.` | https://www.jos.org.cn/ 、CHNDOI |
| 《计算机学报》 | `10.11897/SP.J.1016.` | https://cjc.ict.ac.cn/ 、CHNDOI |
| 《计算机研究与发展》 | `10.7544/issn1000-1239.` | https://crad.ict.ac.cn/ |

**检索套路**：

1. 期刊站内搜：「日志」「微服务」「智能运维」「故障诊断」
2. 优先选 **综述 + 近 5 年 + 与「采集阶段」可对比** 的论文
3. 从目录页复制标题、作者、卷期页，再查 DOI（勿手写 DOI）

**本会话已验证可用的中文条目示例**（可作 curated 模板）：

| cite key | 期刊 | DOI |
|----------|------|-----|
| `limingshu2019logmgmt` | 软件学报 | 10.13328/j.cnki.jos.005601 |
| `jiatong2020logdiag` | 软件学报 | 10.13328/j.cnki.jos.006045 |
| `yuqingyang2022trace` | 软件学报 | 10.13328/j.cnki.jos.006209 |

## 3. DOI 核对（加入 bib 前必做）

### 3.1 三层验证

```bash
# 1) CrossRef（国际 DOI）
curl -s "https://api.crossref.org/works/10.1007/s10664-021-10063-9" | head -c 200

# 2) CHNDOI（中文 DOI，优先于 doi.org）
curl -sI -L "https://www.chndoi.org/Resolution/Handler?doi=10.13328/j.cnki.jos.006045"

# 3) doi.org 落地（出版商可能 403/Cloudflare，不能单凭此否定）
curl -sI -L "https://doi.org/10.1145/3620678.3624652"
```

### 3.2 本会话踩坑

| 反模式 | 后果 | 处理 |
|--------|------|------|
| **手写/记忆 DOI**（如 `zhangjianxun2020` 旧条目） | doi.org **404**，C6 失败 | 删条目或换为已核实文献 |
| 仅看 CNKI 摘要未核 DOI | bib 与官网不一致 | 以期刊目录 + CHNDOI 为准 |
| 将 arXiv DOI 当正式发表 | 审稿质疑 | 正文标明预印本或换期刊版 |

**铁律**：`verify_cited_papers.py` exit 非 0 → **不得**保留该 cite 键投稿。

## 4. 下载技巧

### 4.1 开放 PDF（流水线 A / B 共用）

优先级（与 `download_papers.py` 一致）：

1. `https://arxiv.org/pdf/{id}.pdf`（**直接 URL**，勿依赖 arxiv 库 API，易 429）
2. Unpaywall（`api.unpaywall.org/v2/{doi}`）
3. 论文 JSON 中的 `pdf_url` 字段

本会话 cited 中 **4 篇 PDF** 来源：3× arXiv + 1× Unpaywall（Elsevier OA）。

### 4.2 中文期刊（通常无 OA PDF）

策略：**官网摘要页 HTML 快照** = 合格归档（`archive_status: snapshot`）。

| 来源 | URL 模式 |
|------|----------|
| 软件学报 | `http://www.jos.org.cn/1000-9825/{file_no}.htm`（DOI 末 4–6 位） |
| 计算机研究与发展 | `https://crad.ict.ac.cn/cn/article/doi/{doi}` |

脚本：`verify_cited_papers.py` 自动尝试上述 URL。

### 4.3 闭源英文期刊（Springer / ACM / Wiley）

- `doi.org` 常返回 Cloudflare「Just a moment」，**不能**据此判 DOI 无效
- CrossRef API 返回 200 → DOI **有效**
- 归档：**CrossRef JSON 快照**（`web_snapshots/{key}_crossref.json`）即满足 C6

### 4.4 技术文档 / 书籍

| 类型 | 核实 | 归档 |
|------|------|------|
| 官方文档（OTel、Loki、AWS Blog） | `url` 可访问 | `web_snapshots/*.html` |
| 书籍（SRE、Monitoring） | Open Library / 作者站 | HTML 或 `_meta.json` |
| Leanpub 等易 404 | 换 **Open Library** URL | 更新 bib `url` 字段 |

## 5. 引用门禁一键流程（流水线 B）

```bash
# 1. 写入 latex/references.bib 并在 sections 中 \cite{key}

# 2. 核实 + 下载/快照（约 1–2 min）
python3 scripts/verify_cited_papers.py --download

# 3. 查看汇总
python3 -c "
import json; m=json.load(open('data/papers/cited_papers_manifest.json'))
print('cited', m['cited_count'], m['summary'])
for k,v in m['entries'].items():
    if v.get('archive_status')=='paywall': print('FAIL', k)
"
```

**通过标准**：`summary.paywall == 0` 且脚本 **exit 0**。

## 6. curated.json 手工精选（已知好文献）

搜索 API 不稳定或已明确目标时，直接构造 JSON 供 `download_papers.py` 使用：

```bash
python3 - <<'PY'
import json, pathlib
papers = [
  {"title": "A Comprehensive Survey on Root Cause Analysis in (Micro) Services",
   "authors": ["Tingting Wang", "Guilin Qi"], "year": 2024,
   "doi": "10.48550/arXiv.2408.00803",
   "pdf_url": "https://arxiv.org/pdf/2408.00803.pdf"},
  {"title": "基于日志数据的分布式软件系统故障诊断综述",
   "authors": ["贾统", "李影", "吴中海"], "year": 2020,
   "doi": "10.13328/j.cnki.jos.006045"},
]
pathlib.Path("data/scholar/curated_p3.json").write_text(
    json.dumps(papers, ensure_ascii=False, indent=2))
PY

python .agent/skills/scholar-search/scripts/download_papers.py \
  --input data/scholar/curated_p3.json \
  --output-dir data/papers
```

中文无 PDF 条目放入 curated 后 `download_papers` 可能失败——**属正常**；改由 `verify_cited_papers.py` 抓 HTML 快照。

## 8. 检查清单（后期复用）

> **Cursor Rule**：编辑 bib/正文引用时自动加载 [`.cursor/rules/scholar-cited-checklist.mdc`](../../../.cursor/rules/scholar-cited-checklist.mdc)

- [ ] 新文献 DOI 经 CrossRef 或 CHNDOI 验证
- [ ] 正文 `\cite` 键与 bib 键一致
- [ ] `verify_cited_papers.py --download` exit 0
- [ ] `cited_papers_manifest.json` 中 `paywall: 0`
- [ ] 中文期刊至少有 HTML 快照（非仅「待下载」占位）
- [ ] 无效 DOI 条目已删除或替换（勿留「记忆 DOI」）
- [ ] 扩展阅读 PDF 与 cited 归档已区分（`papers_index` vs `cited_papers_manifest`）
