2. 将本文从“组件实现报告”提升为“分布式定向日志采集框架”，明确网关预筛选、节点定向采集、中心二次过滤三层输入/输出。
3. 同步修改 LaTeX 与 Markdown 主稿；编译成功后保留历史 v4，并另存 v5 时间戳 PDF。
4. 修改后进行二次评审，报告输出到 `docs/20260607-071254-二次评审报告.md`。

## 本轮不直接实施的实验项

| 建议 | 本轮处理方式 | 原因 |
|------|--------------|------|
| 32-64 节点云环境 | 写入后续实验计划 | 当前本地环境为 WSL/Docker 8 节点 |
| DeathStarBench / Alibaba 生产迹 | 写入真实负载计划 | 需要额外数据和环境准备 |
| Promtail+静态过滤端到端对照 | 写入新增基线计划 | 当前已有同架构全量基线，尚未部署 Promtail 对照 |
| OpenTelemetry 尾采样、eBPF 方案直接对比 | 写入基线计划和相关工作互补 | 需要独立采集链路配置 |
| 集群实测消融 | 明确当前图 4 为策略仿真 | 避免过度声称 |
