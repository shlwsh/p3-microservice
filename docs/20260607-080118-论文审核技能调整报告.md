# 论文审核技能调整报告

生成时间：2026-06-07 08:01:18  
技能：`.agent/skills/paper1-multi-agent-review`

## 1. 本次定位

本轮将原有“多角色审稿”技能从一次性评审流程，提升为面向《软件学报》投稿的“评审—改稿—编译—二次评审—文献归档”闭环。核心目标是让技能不仅指出问题，还能依据编辑意见形成可验收的修改计划，并把修改结果落实到主稿、版本化 PDF 和归档报告中。

## 2. 主要调整

1. 增加“基准论文对照与改稿闭环”入口，将包航宇等《智能运维的实践:现状与标准化》和贾统等《基于日志数据的分布式软件系统故障诊断综述》设为固定对照对象。
2. 将编辑建议固化为八维度门槛：结构与逻辑、文献综述、技术创新、实验严谨性、写作呈现、实用性与标准化、算法实现深度、未来工作。
3. 在 DoD 中加入 JOS 专项检查项：国内对比、中文文献覆盖、2023 年以后前沿文献、英文摘要术语一致性、基准论文 A/B 对照。
4. 增加改稿闭环要求：方案文档必须输出到 `docs/` 且文件名带时间戳；成功编译稿件必须带版本号和时间戳；历史稿件不得删除。
5. 增加二次评审要求：修改后至少回归检查八维度建议关闭情况、A/B 互补定位、实验边界、表格溢出、版本化文稿和历史保留。
6. 强化 C6 文献门禁：正文引用必须进入 `latex/references.bib`，并通过 `scripts/verify_cited_papers.py --download` 归档到 `data/papers/`。
7. 明确与 `scholar-search` 技能协同：先检索高相关真实文献，再筛选 DOI/URL 可核验条目，最后通过 manifest 记录 PDF、HTML 或 CrossRef 快照。

## 3. 能力提升

| 能力项 | 调整前 | 调整后 |
|---|---|---|
| 评审深度 | 多角色指出问题 | 可按 JOS 编辑意见做基准论文对照和重大修改裁决 |
| 输出形态 | `reviews/{run_id}/` 评审报告为主 | 同时输出 `docs/` 时间戳方案、二评报告、版本化 PDF |
| 文献核验 | 检查引用缺失 | 要求 DOI/URL 核验与 `data/papers/` 归档 |
| 改稿闭环 | 默认不改稿 | 用户明确要求时可同步改 LaTeX、Markdown、BibTeX 并编译 |
| 接收门槛 | 通用论文质量检查 | 增加 JOS 国内文献、标准化对齐、AIOps 生态融合和可复现性门槛 |

## 4. 本轮新增执行准则

- 参考文献目标从“够用”提升到“50 篇以上、真实可核验、中文英文尽量均衡”。
- 中文文献优先选择《软件学报》《计算机学报》《计算机研究与发展》等国内高水平期刊，尤其关注日志管理、日志诊断、微服务、智能运维和软件智能化方向。
- 英文文献优先选择高引用综述、IEEE/ACM/Elsevier/Springer 期刊会议、arXiv 可下载预印本，以及 OpenTelemetry、Loki、SRE、AWS 等工业一手资料。
- 对无法直接下载 PDF 的真实文献，允许归档官网 HTML、DOI 落地页或 CrossRef JSON 快照，但必须在 manifest 中记录归档方式。

## 5. 后续验收口径

本轮完成后应满足：

1. `latex/sections/zh/*.tex` 中唯一正文引用数不少于 50。
2. `latex/references.bib` 中所有正文引用键均存在。
3. `data/papers/cited_papers_manifest.json` 显示全部正文引用已核实且归档状态为 `ok` 或 `snapshot`。
4. `docs/` 中新增本轮时间戳报告和带版本号 PDF，且旧版本 PDF 保留。
5. PDF 编译日志无 undefined citation/reference，无新增表格溢出问题。
