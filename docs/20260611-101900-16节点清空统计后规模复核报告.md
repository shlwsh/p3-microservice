**日期**：2026-06-11  
**复核目标**：清空 Center/Redis 统计后，仅复跑 16 节点规模矩阵，验证上一轮 `gateway_redis` 增量为 0 的问题是否由历史计数上限或未重置导致。

**结果文件**：`experiments/results/phase4/phase4_20260611_021230.json`

---

## 一、清空方式

复核前执行：

```bash
docker exec redis redis-cli FLUSHALL
docker restart log-center
```

随后确认统计接口归零：

```json
{"gateway_logs_redis":0,"loki_stored":0,"received":0,"filtered":0}
```

说明：`gateway_logs_redis` 来自 Redis 列表长度；`loki_stored`、`received`、`filtered` 来自 Center 进程内存计数。因此清空 Redis 并重启 Center 可清空本轮统计口径。

## 三、复核结论

1. 上一轮 `gateway_redis` 增量为 0 的问题已被确认是统计状态未清空/Redis 列表上限影响，不应作为实验结果使用。
2. 清空统计后，16 节点真实集群每 60 s 产生约 `6703.33±91.78` 条网关流量日志，说明负载确实进入系统。
3. 在相同窗口内，定向模式 Loki 入库为 `48.0±5.2` 条，继续维持低入库量。
4. 关注清单稳定为 32 项，符合 16 节点路由扩展后的预期。
5. Agent 平均 CPU 约 0.03%，Center CPU 均值 0.77%，未表现出资源瓶颈。
