# 32 节点清空统计后规模复核报告

**日期**：2026-06-11  
**复核目标**：与 16 节点复核保持同一口径，清空 Redis 与 Center 统计后，在现有 32 节点 Docker 集群上仅复跑定向模式规模矩阵。  
**执行命令**：`python3 experiments/scripts/phase4_benchmark.py --nodes 32 --skip-quality --skip-extras`  
**结果文件**：`experiments/results/phase4/phase4_20260611_024321.json`

---

## 一、清空方式

复核前执行：

```bash
docker exec redis redis-cli FLUSHALL
docker restart log-center
```

随后确认 32 节点扩展路由可达：

```bash
curl http://localhost:8088/api/service32/status/500
```

该探测请求会使 Center 的 `received/loki_stored` 计数从 0 增至 1，但复核脚本采用每轮 before/after 增量计算，不影响三轮规模矩阵结果。

---

复核采用 32 节点定向模式，60 s 压测，重复 3 次。

| 轮次 | Loki 入库增量 | Gateway Redis 增量 | 关注清单项 | Agent 平均 CPU | Center CPU |

```json
{"gateway_logs_redis":21473,"loki_stored":3103,"received":3103,"filtered":0}
```

该总量包含路由探测、15 s warmup、三轮压测窗口之外的 Agent 后台批量上传或积压刷新，不应用作论文主结果。论文引用应以 `scale.32.runs` 中每轮 before/after 增量为准。

## 三、复核结论

1. **32 节点真实 Docker 集群可稳定运行**：本轮未观察到容器 OOM、重启或 Docker daemon 异常。
2. **定向入库量稳定**：三轮 Loki 入库均为 99 条，标准差为 0。
3. **负载确实进入系统**：每 60 s 网关流量日志增量为 `6692.67±121.88` 条。
4. **关注清单达到当前上限**：32 节点下关注清单稳定为 50 项，已触达 `Top-K=50` 配置上限。
5. **资源开销仍低**：Agent 平均 CPU 为 `0.06±0.01%`，Center CPU 为 `0.06±0.07%`。
