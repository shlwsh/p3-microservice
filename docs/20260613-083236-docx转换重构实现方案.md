# DOCX 转换重构实现方案

生成时间：2026-06-13 08:32:36

## 1. 问题分析

原 `scripts/build_docx.sh` 采用“LaTeX 预处理 → Pandoc → reference-doc”的通用转换链。该方案对 `rjthesis.cls` 的首页题头、摘要、引用格式、附中文参考文献、作者简介、算法环境、图表位置和自定义字体命令缺乏结构化理解，且预处理阶段会删除或重组部分前置区内容，导致 DOCX 相比 `latex/main-jos.pdf` 出现内容丢失和排版失真。

本轮改造将转换模型从“让 Pandoc 猜测 LaTeX 语义”调整为“按《软件学报》文稿结构显式生成 WordprocessingML”。

## 2. 格式依据

新实现严格读取并使用以下格式定义：

- `docs/format/jos_2025_docx_format_definitions.json`
- `docs/format/jos_2025_docx_format_definitions.md`

当前 DOCX 的页面设置来自上述 JSON：

- 纸张：18.40 cm × 26.00 cm，对应 `w=10433, h=14742`
- 页边距：上 1.00 cm、右 1.45 cm、下 2.20 cm、左 1.45 cm
- 页眉/页脚距：页眉 1.30 cm、页脚 1.00 cm
- 分栏：单栏
- 首页独立页眉页脚：启用 `titlePg`

样式实现参照抽取报告中的关键样式：正文 `145`、中文题名 `64`、作者 `65`、单位 `66`、中文摘要 `117`、关键词 `118`、英文引用格式 `121`、一级标题 `213`、二级标题 `215`、参考文献标题 `126`、参考文献正文 `129`。

## 3. 新流水线

入口仍为：

```bash
bash scripts/build_docx.sh
```

执行步骤：

1. 检查 `python3`、`Pillow/PIL`、`pdftotext`、`docs/format` JSON 和 `latex/main-jos.pdf`。
2. 调用 `scripts/build_jos_docx.py`：
   - 解析 `latex/main-jos.tex` 和 `latex/sections/zh/*.tex`。
   - 展开摘要、关键词、正文、图、表、算法、公式、参考文献和作者简介。
   - 使用 `.bbl` 生成 51 条英文参考文献，并保留主稿附中文参考文献。
   - 使用 `figures/*.png` 嵌入 8 张图。
   - 直接写出 DOCX ZIP/XML，不依赖 Pandoc 或 `python-docx`。
3. 调用 `scripts/verify_jos_docx.py`：
   - 从 DOCX XML 提取文本、表格、图片、页面设置。
   - 用 `pdftotext` 抽取 `latex/main-jos.pdf` 文本。
   - 对关键章节、图表、算法、参考文献、作者简介和邮箱做覆盖检查。
   - 校验 DOCX 页面 XML 与 `docs/format` JSON 完全一致。
4. 版本化输出到 `docs/`。

额外复核：同一 DOCX 与生成前已有的版本化 PDF `docs/v53-论文稿件-jos-20260613-082458.pdf` 也通过一致性校验，DOCX/PDF 文本字符比例为 0.976。

关键指标：

| 项 | 结果 |
|---|---:|
| 表格对象数 | 6 |
| 编号论文表格 | 5 |
| 图片数 | 8 |
| 英文参考文献条目数 | 51 |
| 关键内容标记覆盖 | 22/22 |
| DOCX/PDF 文本字符比例 | 0.976 |

| 页边距 XML | 与 `docs/format` 一致 |

表格对象数为 6 的原因是正文除 5 个编号 `table` 环境外，算法节还包含一个无编号的 `center/tabular` 三次转换说明；新 DOCX 保留该结构以避免内容丢失。
