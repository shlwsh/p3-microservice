## 四、不宜直接写入或需加限制的结论

以下内容不能直接作为强结论：

1. **`gateway_redis` 增量为 0**  
   本轮 JSON 中 8/16 节点 `delta_gateway_redis` 均为 0，但 Center 实时接口曾显示 `gateway_logs_redis=100000`。这说明该计数可能已达到上限或没有按轮次重置，因此不应用本轮 `gateway_redis` 增量论证流量规模。

2. **即时探针漏报率偏高**  
   虽然 `attention_covers_500=true`，但即时探针漏报率为 76.67%，稳态探针漏报率为 75.0%。这更像是批处理窗口、计数轮询方式或 Agent 上传节奏导致的“即时可见性”问题，不能写成规则级漏报率。论文中必须继续区分“规则级覆盖”和“即时探针命中”。

3. **16 节点入库量与 8 节点相同**  
   这可能说明 Top-K 清单和中心过滤有效，也可能与统计窗口、去重键或 Center 计数方式有关。进入摘要前建议追加一次“只跑 16 节点、清空/重启指标后”的规模复核。
