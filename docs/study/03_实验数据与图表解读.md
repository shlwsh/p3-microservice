## 2. 主对比表（三期，论文表1）

数据源：`phase3_latest.json` → `comparison` 字段（均值 ± 标准差，3 轮重复）

| 指标 | 定向采集 | 全量采集 | 降低比例 |
|------|---------|---------|---------|
| **Loki 入库 (条/轮)** | **72 ± 0** | **4388 ± 1** | **98.4%** |
| Agent CPU (%) | 0.05 ± 0.01 | 0.08 ± 0.02 | 37.5% |
| Agent 内存 (MB) | 95.6 ± 4.2 | 97.5 ± 1.7 | 2.0% |
| 关注清单模式数 | 16 | — | — |
| 网关 Redis 增量 | ~19700 条/轮 | — | — |

### 解读要点

1. **主 claim** 是 Loki 入库条数降幅，不是 CPU（CPU 降幅有限因 Agent 本身轻量）。
2. 全量标准差 **σ = 1**（审核 P0：勿写成 ±1000）。
3. 定向模式空清单时仅采 ERROR+（`rule_matcher.go` 修复），是全量差距拉大的关键因素之一。
4. **公平性披露**（审核 P1）：定向 Agent 日志发射间隔 **2 s**，全量模式 **400 ms**——须在 §6 正文说明。

## 9. 常见误读（写入论文时规避）

| 错误 | 正确 |
|------|------|
| 把首期 68% 当最终结论 | 主结论用三期 **98.4%** |
| 全量 σ 写 ±1000 | **±1**（L0 字段 `full.loki_stored_stdev`） |
| 不说发射频率差异 | 定向 2 s vs 全量 400 ms 须在 §6 披露 |
| 消融当实测 | 图4 标注「仿真模型」 |
| CPU 降幅当主指标 | 主指标是 **Loki 入库条数** |
