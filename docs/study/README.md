## 电梯演讲（30 秒）

微服务日志若全量采集，CPU、带宽与 Loki 存储压力都很大。我们做一个**分布式定向日志采集组件**：网关先看流量谁慢、谁错，中心动态生成**关注清单**，各 Sidecar Agent **只采相关日志**，经 gRPC 上传后在 Center 二次过滤写入 Loki。三期八节点实测：较全量采集 **Loki 入库减少 98.4%**（72 vs 4388±1 条/轮）。
