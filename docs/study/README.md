## 电梯演讲（30 秒）

微服务日志全量采集会把大量低价值日志推入中心端，带来带宽、存储和排障成本。本文提出**分布式定向日志采集框架**：网关先识别慢请求、错误码和异常 URL，Center 生成动态关注清单，Sidecar Agent 只采与清单匹配的应用日志，最后由 Center 二次过滤后写入 Loki。八节点、180 s、并发 50、每模式 3 轮实测显示，相较同架构全量采集，Loki 入库量降低 **98.4%**（72 条 vs 4388±1 条），Agent CPU 降低 **37.5%**。

当前 v9 稿件已完成两轮顶刊规范复审：实验节补 RQ1–RQ4 映射，退避公式已编号，建议引用格式和基础作者简介已补齐。正文引用文献为 **51 篇**，`data/papers/cited_papers_manifest.json` 显示 `ok=51, failed=0, paywall=0, doi_valid=51`。
