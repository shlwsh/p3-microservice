# 实验方案

> 微服务架构下分布式定向日志采集组件 — 实验设计  
> 对应科研计划阶段：首期（仿真）→ 二期（实测）→ 三期（规模化）

---

## 1 实验目标

验证以下假设：

1. **H1**：定向采集较 Loki 全量采集降低节点 CPU/内存 ≥ 40%
2. **H2**：定向采集降低网络带宽消耗 ≥ 60%
3. **H3**：定向采集减少入库日志量 65%–80%（业务异常占比依赖）
4. **H4**：固定缓存块 + 指数退避使中心故障时丢失率 < 1%
5. **H5**：关注清单生成在万级流量日志下耗时 < 100 ms

## 4 消融实验（A1–A4）

| 编号 | 配置 | 预期 |
|------|------|------|
| A1 | `ATTENTION_LIST=off` | 日志量→全量 |
| A2 | `CACHE_BLOCK=off` | 内存飙升、OOM 风险 |
| A3 | `BACKOFF=off` | 丢失率上升、惊群 |
| A4 | `PRESSURE_AWARE=off` | 高负载 CPU 波动 |

首期以仿真数据呈现趋势（见 `experiments/results/phase1/`），二期通过环境变量或 compose profile 切换。

## 5 可靠性实验（R1–R2）

### R1 网络抖动

```bash
# 在网关网络命名空间注入延迟（需 root）
sudo tc qdisc add dev eth0 root netem delay 100ms 20ms loss 1%
```

指标：上传成功率、P99 延迟、BoltDB 兜底条数。

### R2 中心宕机

```bash
docker stop log-center
# 持续写入 5 min 后恢复
docker start log-center
```

指标：本地兜底量、恢复后补传成功率。

## 7 统计与报告规范

1. 每个实验 ≥ 3 次重复，报告均值 ± 标准差
2. 对比实验需固定负载种子（`seed=42`）
3. 结果 JSON 存入 `experiments/results/{phase}/`
4. 图表仅通过 `figures/plot_*.py` 生成，不手工 P 图
5. 论文中首期仿真数据须标注「仿真」；实测后替换并更新图注
