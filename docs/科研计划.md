## 1 研究定位

### 1.1 论文题目

**微服务架构下分布式定向日志采集组件的设计与实现**

### 1.2 科学问题

| 编号 | 科学问题 | 当前证据 | 后续补强 |
|------|----------|----------|----------|
| RQ1 | 能否基于网关流量日志动态生成关注清单，并显著降低日志入库量？ | 三期：72 vs 4388±1 条；四期 8/16 节点规模矩阵 | 32/64 云环境实测 |
| RQ2 | 定向采集是否能控制 Sidecar 资源开销？ | CPU 降低 37.5%；16 节点 Agent CPU 0.02% | 时间序列 P95/P99 |
| RQ3 | 三次策略转换能否贯通网关、节点与中心链路？ | 端到端已打通；中心恢复约 1.1 s | K8s 多租户 |
| RQ4 | 固定缓存块与压力感知退避能否支撑可靠传输？ | 仿真消融 + 单测 | 集群实测消融 |

### 1.3 投稿固定贡献

1. **框架贡献**：网关预筛选、节点定向采集、中心二次过滤三层框架。  
2. **算法贡献**：关注清单生成、策略三次转换、固定缓存块、压力感知退避。  
3. **验证贡献**：三期八节点对比 + 四期规模/质量/基线扩展，入库量降低 98.4%。

### 1.4 研究边界

| 维度 | 本文已做 | 本文不声称已完成 |
|------|----------|------------------|
| 场景 | HTTP 网关 + Apache Combined 风格日志 | 跨语言二进制协议全量适配 |
