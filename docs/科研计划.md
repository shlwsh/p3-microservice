## 1 研究定位

### 1.1 论文题目

**微服务架构下分布式定向日志采集组件的设计与实现**

### 1.2 科学问题

| 编号 | 科学问题 | 当前证据 | 后续补强 |
|------|----------|----------|----------|
| RQ1 | 能否基于网关流量日志动态生成关注清单，并显著降低日志入库量？ | 八节点三期实测：72 条 vs 4388±1 条，降低 98.4% | 多规模 16/32/64 节点与真实负载 |
| RQ2 | 定向采集是否能控制 Sidecar 资源开销？ | Agent CPU 降低 37.5%，内存降低 2.0% | 补充时间序列 CPU/内存与 P95/P99 |
| RQ3 | 三次策略转换能否贯通网关、节点与中心链路？ | Nginx 共享内存 → Agent gRPC → Redis → 关注清单生成已打通 | Kubernetes 部署和多租户策略隔离 |
| RQ4 | 固定缓存块与压力感知退避能否支撑可靠传输？ | 策略仿真消融、微基准和 Go 单元测试 | 集群实测消融、中心故障恢复时间 |

### 1.3 投稿固定贡献

1. **框架贡献**：提出网关预筛选、节点定向采集、中心二次过滤三层组成的分布式定向日志采集框架，补足 AIOps 数据链路中的采集前端减量环节。
2. **算法贡献**：给出关注清单动态生成、定向策略三次转换、固定缓存块和压力感知指数退避的形式化描述与 Go 实现。
3. **验证贡献**：在 WSL/Docker 八节点原型上完成端到端验证、三期重复对比实验、策略仿真消融与算法微基准测试，实测日志入库量降低 98.4%。

### 1.4 研究边界

| 维度 | 本文已做 | 本文不声称已完成 |
|------|----------|------------------|
| 场景 | HTTP 网关 + Apache Combined 风格应用日志 | 跨语言二进制协议全量适配 |
| 系统 | Go + gRPC + Redis + Loki 原型 | 生产级商业 APM 全功能替代 |
| 实验 | 八节点 180 s、n=3 原型实测 | 32/64 节点生产级结论 |
| 基线 | 同架构全量采集 | Promtail/OTel/eBPF 已端到端实测 |
