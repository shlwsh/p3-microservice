## 1 研究定位

### 1.1 论文题目（拟定）

**微服务架构下分布式定向日志采集组件的设计与实现**

### 1.2 科学问题

| 编号 | 科学问题 | 验证方式 |
|------|---------|---------|
| SQ1 | 能否基于网关流量日志动态生成关注清单，在可接受信息损失下显著降低日志采集量？ | 对比实验 E1–E3 |
| SQ2 | 固定缓存块与压力感知退避能否在节点资源受限时保障传输可靠性？ | 消融 A2/A3 + 重试验证 |
| SQ3 | 定向策略三次转换能否贯通网关预筛、节点过滤与中心入库全链路？ | 架构验证 + 端到端演示 |

### 1.3 三条贡献（投稿固定表述）

1. **架构贡献**：提出主从式分布式定向日志采集架构，将网关流量感知、节点侧过滤与中心侧二次处理解耦，适配 Sidecar 部署。
2. **算法贡献**：给出关注清单动态生成（O(N log N)）、固定缓存块、定向策略三次转换及压力感知指数退避的形式化描述与 Go 实现。
3. **验证贡献**：在 WSL/Docker 原型集群上完成首期对比、消融与微基准实验，验证资源消耗与日志量显著下降的可行性。

### 1.4 研究边界

| 维度 | 本文做 | 本文不做 |
|------|--------|---------|
| 场景 | 微服务 HTTP 网关 + 应用日志 | 跨语言二进制协议全量适配 |
| 存储 | Grafana Loki | 自研时序存储 |
| 部署 | Docker Compose / WSL 原型 | 生产级 K8s 多租户运维 |
| 对比基线 | Loki 全量采集 / Promtail 模式 | 商业 APM 全功能对比 |
