# 部署说明

本文档描述在 **WSL2 + Docker Desktop** 环境下部署本项目的完整流程，包括环境要求、配置步骤、验证方法与常见问题排查。

> 架构设计与实现细节见 [v2-实现方案.md](./v2-实现方案.md) 第 5 章。

---

## 1 环境要求

### 1.1 硬件与系统

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10/11 + WSL2 | Windows 11 22H2+ |
| Linux 发行版 | Ubuntu 22.04+ | Ubuntu 24.04 |
| 内存（WSL 分配） | 6 GB | 8 GB+ |
| CPU | 2 核 | 4 核+ |
| 磁盘 | 20 GB 可用空间 | SSD |

### 1.2 软件依赖

| 软件 | 版本 | 说明 |
|------|------|------|
| Docker Desktop | 4.x+ | 需启用 WSL2 集成 |
| Docker Compose | 2.x+ | 随 Docker Desktop 附带 |
| Go（可选） | 1.22+ | 仅混合开发模式需要 |

### 1.3 代码存放位置

- **推荐**：`~/work/p3-microservice`（WSL 原生 ext4 文件系统，构建与 I/O 性能最佳）
- **不推荐**：`/mnt/c/Users/...`（跨文件系统挂载，Docker 构建明显变慢）

## 2 部署前配置

按顺序完成以下配置，再进行集群启动。

### 2.1 安装并启用 Docker Desktop

1. 安装 [Docker Desktop for Windows](https://docs.docker.com/desktop/setup/install/windows-install/)
2. 打开 **Settings → Resources → WSL Integration**
3. 勾选当前使用的 WSL 发行版（如 Ubuntu）
4. 确认 Docker Desktop 托盘图标处于 Running 状态

在 WSL 终端验证：

```bash
docker --version
docker compose version
docker info
```

### 2.2 配置 WSL 内存（推荐）

将 `deploy/wsl/.wslconfig.example` 复制到 Windows 用户目录：

```
C:\Users\<用户名>\.wslconfig
```

示例内容：

```ini
[wsl2]
memory=8GB
processors=4
swap=2GB

[experimental]
autoMemoryReclaim=gradual
```

修改后**必须**执行 `wsl --shutdown`，再重新打开 WSL 终端使配置生效。

### 2.3 修复 Docker 凭证助手（WSL 必做）

WSL 下首次拉取镜像可能报错：

```
error getting credentials - err: fork/exec /usr/bin/docker-credential-desktop.exe: exec format error
```

**自动修复**（推荐）：`setup.sh start` 会自动处理。

**手动修复**：编辑 `~/.docker/config.json`：

```json
{
  "auths": {},
  "credsStore": ""
}
```

### 2.4 端口占用检查

集群默认占用以下宿主机端口，启动前确认无冲突：

| 端口 | 服务 |
|------|------|
| 8088 | Gateway（Nginx） |
| 8080 | Center HTTP API |
| 9091 | Center gRPC（对外映射） |
| 3000 | Grafana |
| 3100 | Loki |
| 6379 | Redis |
| 9090 | Prometheus |

```bash
ss -tlnp | grep -E '8088|8080|3000|3100|6379|9090'
```

> **说明**：网关使用 **8088** 而非 80，因为 Windows 上 IIS 常占用 80 端口，WSL 访问 `localhost:80` 会命中 IIS 而非容器。

## 3 部署步骤

### 3.1 一键部署（推荐）

```bash
# 进入项目根目录
cd ~/work/p3-microservice

# 环境检查（不启动服务）
./deploy/wsl/setup.sh check

# 构建并启动完整集群（含 WSL 资源优化）
./deploy/wsl/setup.sh start

# 健康检查
./deploy/wsl/verify.sh
```

`setup.sh` 支持的命令：

| 命令 | 说明 |
|------|------|
| `check` | 仅环境检查 |
| `start` | 检查 + 启动（启用 `docker-compose.wsl.yml` 内存限制） |
| `start-full` | 检查 + 启动（不启用内存限制，适合 16GB+ 环境） |
| `stop` | 停止并移除容器 |
| `logs` | 跟踪 Center 日志 |
| `verify` | 运行健康检查 |

### 3.2 手动部署

```bash
cd deploy/docker
docker compose -f docker-compose.yml -f docker-compose.wsl.yml up -d --build
```

内存充裕时，可省略 WSL 覆盖文件：

```bash
docker compose -f docker-compose.yml up -d --build
```

### 3.3 启动的容器清单

| 容器名 | 镜像/构建 | 作用 |
|--------|----------|------|
| `loki` | grafana/loki:2.9.4 | 日志存储 |
| `grafana` | grafana/grafana:10.3.3 | 可视化 |
| `redis` | redis:7-alpine | 高速缓存 |
| `prometheus` | prom/prometheus:v2.50.1 | 指标采集 |
| `log-center` | 本地构建 | 日志中心（Go） |
| `gateway` | openresty/openresty | Nginx 网关 + Lua 流量采集 |
| `gateway-agent` | 本地构建 | 网关 Sidecar Agent |
| `service-1` / `service-2` | httpbin | 模拟微服务 |
| `service-1-agent` / `service-2-agent` | 本地构建 | 微服务 Sidecar Agent |

## 4 配置文件说明

项目采用**配置分层**：本地调试与 Docker 部署使用不同配置文件。

### 4.1 配置文件对照

| 文件 | 场景 | 关键地址 |
|------|------|---------|
| `agent/configs/agent.yaml` | 本地 `go run` | `center_address: localhost:9091` |
| `center/configs/center.yaml` | 本地 `go run` | `redis: localhost:6379`, `loki: localhost:3100` |
| `deploy/docker/center-config.yaml` | Center 容器 | `redis:6379`, `loki:3100`（Docker 网络服务名） |
| `deploy/docker/gateway-agent-config.yaml` | 网关 Sidecar | `center:9090` |
| `deploy/docker/service-agent-config.yaml` | 服务 Sidecar | `center:9090` |
| `deploy/docker/prometheus.yml` | Prometheus 采集目标 | — |
| `deploy/docker/docker-compose.wsl.yml` | WSL 内存限制覆盖 | — |

### 4.2 地址规则

- **容器间通信**：使用 Docker Compose 服务名（如 `center`、`redis`、`loki`）
- **宿主机访问容器**：使用 `localhost` + 映射端口（见下表）
- **Center gRPC**：容器内 `9090`，宿主机映射为 `9091`（避免与 Prometheus `9090` 冲突）

### 4.3 端口映射

| 服务 | 宿主机端口 | 容器端口 | 用途 |
|------|-----------|---------|------|
| Gateway | **8088** | 80 | HTTP 网关入口 |
| Center HTTP | 8080 | 8080 | 管理 API / 健康检查 |
| Center gRPC | **9091** | 9090 | Agent 通信（宿主机侧） |
| Loki | 3100 | 3100 | 日志 Push/Query API |
| Grafana | 3000 | 3000 | Web 控制台 |
| Redis | 6379 | 6379 | 缓存 |
| Prometheus | 9090 | 9090 | 指标查询 |
| Agent Metrics | 9100 | 9100 | Sidecar 指标（共享网关网络栈） |

## 5 验证与使用

### 5.1 健康检查

```bash
./deploy/wsl/verify.sh
```

预期输出 6 项全部通过：Gateway、Center、Loki、Grafana、Prometheus、Redis。

也可手动验证：

```bash
curl http://localhost:8088/health
curl http://localhost:8080/api/v1/health
curl http://localhost:3100/ready
```

### 5.2 产生测试流量

```bash
# 正常请求
curl http://localhost:8088/api/service1/get

# 模拟错误
curl http://localhost:8088/api/service2/status/500

# 查看已注册 Agent
curl http://localhost:8080/api/v1/agents
```

### 5.3 访问 Web 控制台

| 服务 | 地址 | 默认账号 |
|------|------|---------|
| Grafana | http://localhost:3000 | admin / admin |
| Prometheus | http://localhost:9090 | — |
| Center API | http://localhost:8080 | — |

### 5.4 配置 Grafana Loki 数据源

首次使用需在 Grafana 中添加数据源：

1. 打开 http://localhost:3000 ，使用 admin / admin 登录
2. 进入 **Connections → Data Sources → Add data source**
3. 选择 **Loki**，URL 填写 `http://loki:3100`
4. 点击 **Save & Test**

## 7 混合开发模式

修改 Go 代码后快速迭代，无需每次重建镜像。

**步骤 1**：仅启动基础设施

```bash
cd deploy/docker
docker compose up -d loki grafana redis prometheus
```

**步骤 2**：本地运行 Center

```bash
cd center
export CENTER_CONFIG_PATH=./configs/center.yaml
go run ./cmd/center/
```

**步骤 3**：本地运行 Agent

```bash
cd agent
export AGENT_CONFIG_PATH=./configs/agent.yaml
go run ./cmd/agent/
```

本地配置中 `localhost` 地址对应 Docker 映射端口；Center gRPC 使用 `localhost:9091`。

## 8 资源预算

启用 `docker-compose.wsl.yml` 后，容器内存上限合计约 **2 GB**：

| 服务 | mem_limit |
|------|-----------|
| Loki | 512 MB |
| Grafana | 256 MB |
| Redis | 192 MB |
| Prometheus | 256 MB |
| Center | 256 MB |
| 每个 Agent | 128 MB |

8 GB WSL 环境下推荐使用 `setup.sh start`；16 GB 以上可使用 `setup.sh start-full`。

## 9 常见问题排查

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| `docker: command not found` | 未安装或未启用 WSL 集成 | 安装 Docker Desktop，Settings → WSL Integration 勾选当前发行版 |
| `Cannot connect to the Docker daemon` | Docker Desktop 未启动 | 启动 Docker Desktop，等待就绪 |
| `docker-credential-desktop.exe: exec format error` | WSL 凭证助手不兼容 | 运行 `setup.sh start` 自动修复，或手动修改 `~/.docker/config.json` |
| `localhost:80` 返回 IIS 404 | Windows IIS 占用 80 | 使用 `http://localhost:8088` |
| Gateway 容器 Exited (1) | Nginx 日志路径问题 | `docker compose up -d --force-recreate gateway` |
| 构建失败 `missing go.sum entry` | 依赖未锁定 | `docker run --rm -v $(pwd)/agent:/app -w /app golang:1.22-alpine go mod tidy`（center 同理） |
| 端口被占用 | 宿主机已有服务监听 | `ss -tlnp \| grep :8088` 查进程，或修改 compose 端口映射 |
| 构建/启动极慢 | 代码在 `/mnt/c/` | 移至 WSL 原生路径如 `~/work/` |
| 容器 OOM 重启 | WSL 内存不足 | 增大 `.wslconfig` memory，或使用 `docker-compose.wsl.yml` |
| Loki `/ready` 返回 503 | 服务尚在启动 | 等待 10–20 秒后重试 |
| Grafana 无法查日志 | 未配置数据源 | 添加 Loki 数据源 `http://loki:3100` |
| Agent 连不上 Center | 地址配置错误 | 容器内 `center:9090`，宿主机调试 `localhost:9091` |
| `network_mode: service:gateway` 失败 | gateway 未运行 | 先确保 gateway 正常，再启动 gateway-agent |
